📋 工單自動化 使用說明書

KKday BE2 工單自動化工具 · 操作指南

⚙️ 帳號設定
🔐
BE2 帳號 / 密碼
填入你的 BE2 登入帳號與密碼。勾選「記住帳號與名字」後，重整或重開頁面時會自動帶入（密碼不會儲存，每次需重新輸入）。
👤
你的名字
填入你的名字，將顯示在訂單備註紀錄上，方便追蹤是誰執行的。
📑 功能分頁總覽
🟡
挽單 挽單
輸入訂單編號（或供應商編號），自動建立挽單工單、後拋，並填入挽單通知。支援多筆同時執行，每行一筆。
額滿：工單分類 訂單異動 → 額滿 → 挽單，可選依商品頁面或依OP提供。
未成團：工單分類 訂單異動 → 供應商通知 → 改期，固定依OP提供。
🟢
一般工單 一般工單
輸入訂單編號，選擇工單分類（大 / 中 / 小），填入對客通知內容，自動建立工單並後拋。支援多筆同時執行，每行一筆。
🔴
對客通知 行前通知
自動建立對客通知工單並發送。分為兩種模式：
統一通知：多筆訂單發送同一份通知內容，填法與其他頁相同。
個別通知：每筆訂單發送不同內容，需上傳 Excel 範本（見下方說明）。
✏️ 訂單編號填寫方式
📋
KKday 訂單編號 vs 供應商編號
兩欄填一即可，每行對應一筆訂單。若同時填兩欄，系統會將同一行的 KKday 編號與供應商編號視為同一筆訂單。

範例（多筆）：
26KK231600728
25KK258710416
25KK269680760
💡 挽單 / 一般工單若輸入供應商編號，系統會自動查詢對應的 KKday 訂單編號再繼續執行。
📂 批次對客通知（各單不同內容）
1️⃣
下載範本
點擊「⬇ 下載範本」取得 Excel 範本。A 欄填 KKday 訂單編號，B 欄填各單通知內容（中英版本或純英文）。
2️⃣
選擇 Excel 上傳
點擊「📂 選擇 Excel」上傳填好的檔案，頁面會顯示「已解析 N 筆資料」確認筆數無誤。
3️⃣
開始批次發送
確認筆數後，點擊「▶ 開始批次發送」，系統依序處理每筆訂單。
🔀 挽單類型 & 可改類型 挽單專屬
📋
額滿
工單分類：訂單異動 → 額滿 → 挽單。後拋備註自動填入「額滿挽單」。
可改類型有兩種選項：
 ・依商品頁面：自動讀取商品頁語系，套用對應語言通知模板。建議使用。
 ・依 OP 提供：出現「OP 可改場次」輸入欄，填入 OP 提供的可改日期。
📋
未成團
工單分類：訂單異動 → 供應商通知 → 改期。後拋備註自動填入「未成團挽單」。
可改類型固定為依 OP 提供（自動選取），請在「OP 可改場次」欄填入可改日期。
📊 執行結果
✅
成功
顯示綠色「成功」標籤，並附上工單 ID、最晚處理時間等資訊。點擊「展開詳情」可查看完整執行步驟紀錄。
❌
失敗
顯示紅色「失敗」標籤，點擊「展開詳情」可查看失敗原因與執行到哪個步驟出錯。
⬇️
匯出 Excel
執行完成後，右上角出現「⬇ 匯出 Excel」按鈕，可將本次執行的所有訂單結果（訂單編號、工單 ID、狀態、最晚處理時間等）下載為 Excel 檔案。
💡 切換分頁後，各頁的執行結果不會消失，切回去仍可查看。
⏹ 停止執行
🛑
如何停止 ⏹ 停止
執行中時，頁面頂部右側會出現紅色「⏹ 停止」按鈕。點擊後，系統會在當前步驟完成後立即中斷（最多等待幾秒），並不再繼續處理後續訂單。

已中斷的訂單結果會顯示「失敗 / 已手動停止」，後續未處理的訂單則不會出現在結果中。
⚠️ 停止後，已完成發送的訂單不會被撤回，只有尚未開始的訂單會被跳過。
🕓 歷史紀錄
📜
查看過往執行紀錄
頁面下方的「歷史紀錄」區塊，會依目前所在分頁自動篩選對應的紀錄：
• 挽單頁：顯示挽單紀錄（額滿 & 未成團）
• 一般工單頁：顯示一般工單紀錄
• 對客通知頁：顯示行前通知紀錄

每筆紀錄包含執行時間、訂單編號、工單 ID 及執行結果。
⚠️ 注意事項
• 執行期間請勿關閉或重新整理頁面，否則執行中的訂單會被強制中斷。
• 系統最多同時執行 2 個瀏覽器，多筆訂單時會自動排隊（頂部顯示「執行中 N/2」）。
• 密碼不會被儲存，每次使用需重新輸入。
• 若出現「連線失敗」，請確認後端服務是否正常運行（右上角顯示「服務運作中」為正常）。
